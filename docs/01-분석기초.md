# 퍼포먼스 인사이트 분석 기초 프레임워크
> 퍼포먼스 인사이트 분석의 사고 구조와 실행 원칙을 정의한 기준 문서.
> 모든 파트너사 분석에 공통 적용되며, 클라이언트별 설정값은 별도 설정 파일에서 불러온다.

---

## 원칙 1 — 단일 KPI 기준 (Single KPI Standard)

**모든 분석은 하나의 KPI를 기준으로 처음부터 끝까지 일관되게 흐른다.**

- 리포트 내 모든 섹션(매체, 캠페인, 소재)에서 동일한 KPI로 성과를 측정한다.
- KPI가 중간에 바뀌거나 섞이면 분석의 인과관계가 끊긴다. 절대 혼용 금지.
- 단일 KPI는 파트너사와 사전 합의 후 클라이언트 설정 파일에 명시한다.
- 보조 지표(CTR, CPC, 노출수 등)는 KPI의 변화를 **설명하는 근거**로만 사용한다.

> **예시:** KPI = CPL(리드당 비용)이면, "CTR이 높아졌다"는 정보는 "CPL이 어떻게 변했는지"를
> 설명할 때만 의미가 있다. CTR 자체가 성과가 되면 안 된다.

---

## 원칙 2 — 펀넬 분해 순서 (Top-Down Breakdown)

**분석은 반드시 큰 퍼널에서 작은 단계로 쪼개어 내려간다.**

    [LEVEL 0] 전체 통합 성과
         |
    [LEVEL 1] 매체별 (Google / Naver / Meta 등)
         |
    [LEVEL 2] 캠페인별
         |
    [LEVEL 3] 광고그룹별
         |
    [LEVEL 4] 키워드 및 소재 유형별
         |
    [LEVEL 5] 키워드 및 소재 내용별 (실제 텍스트/이미지 단위)
         |
    [LEVEL 6] 확장소재 및 외부 요인 점검
              (사이트링크, 콜아웃, 리뷰, 계절성, 경쟁 변화 등)

**분해 원칙:**
- 상위 레벨에서 이상 신호가 감지될 때만 하위 레벨로 내려간다.
- 전체 성과가 양호할 때 소재 단위까지 분석하면 인사이트가 아닌 노이즈가 된다.
- 각 레벨에서 "KPI 기준으로 상위/하위 N개"를 먼저 추출한다.
- 분해 종료 조건: 해당 레벨에서 원인이 특정되면 그 아래는 생략 가능.

---

## 원칙 3 — 3단계 분석 시퀀스 (Insight Sequence)

**인사이트 하나마다 반드시 아래 3단계를 순서대로 거친다.**

### STEP 1. 팩트 정의 (What happened — 명확하고 데이터 기반)

- "무엇이 변했는지"를 숫자로 정의한다.
- **절대 단정 금지:** 하나의 변수가 결과를 만들었다고 단정하지 않는다.
- **비교 기준 명시:** 전주 대비 / 전월 대비 / 목표 대비 중 어느 기준인지 항상 명시한다.
- 긍정·부정 모두 동일한 팩트 정의 방식을 따른다.

  좋은 예: "3주차 전체 CPL이 전주 대비 23% 상승 (32,000원 → 39,400원)"
  나쁜 예: "경쟁사 증가로 CPL이 올랐다" (원인을 팩트처럼 서술)

### STEP 2. 원인 파악 (Why it happened — 임팩트 순서)

- 원인은 하나만 있을 수 없다는 것을 전제한다.
- **임팩트 크기 순**으로 나열한다. 가장 영향이 큰 원인이 1순위.
- 각 원인에는 이를 뒷받침하는 데이터 근거를 반드시 붙인다.
- 추정 원인과 데이터 기반 원인을 구분해서 표기한다.
  - 데이터 기반: [확인됨]
  - 추정/가설:   [추정]

  예시:
  [확인됨] Google SA CPL +41% 상승이 전체 CPL 악화의 주요인 (전체 비중 68%)
  [추정]   경쟁 입찰가 상승으로 CPC 증가한 것으로 추정 (Auction Insights 데이터 부재)

### STEP 3. 액션 제시 (What to do — 가장 임팩트 큰 것 먼저)

- STEP 2에서 파악된 원인 중 가장 임팩트가 큰 이슈에 대한 해결책을 1순위로 제시한다.
- 액션은 "즉시 실행 가능한 것"과 "중장기 개선"으로 분리한다.
- 불확실한 원인에 대한 액션은 "검증 후 실행" 형태로 제안한다.

  [즉시]     특정 광고그룹의 입찰가 10% 하향 조정
  [검증 후]  Auction Insights 확인 후 경쟁사 대응 전략 수립
  [중장기]   전환율 낮은 키워드 그룹 재구성

## 분석 태도 원칙 (Anti-Pattern 방지)

| 금지 패턴 | 올바른 접근 |
|---|---|
| "이번 주는 경쟁사 때문에 CPL이 올랐습니다" | 경쟁 요인은 [추정]으로 표기, 데이터로 뒷받침 필요 |
| "소재 A가 좋아서 CTR이 높습니다" | KPI(CPL/ROAS)와의 연결 없이 보조 지표만 언급 금지 |
| "전반적으로 성과가 좋습니다" | 어떤 KPI가, 얼마나, 무엇과 비교해서 좋은지 수치로 명시 |
| 부정 성과는 깊게, 긍정 성과는 얕게 | 긍정·부정 모두 동일한 분석 깊이 적용 |
| 가장 눈에 띄는 지표부터 분석 | 반드시 전체 통합(LEVEL 0)부터 시작 |

## 인사이트 문장 작성 가이드

**구조:** [비교 기준] + [팩트 수치] + [원인 1순위] + [액션]

예시 (좋음):
  "2주차 전체 CPL이 전주 대비 18% 상승했습니다 (28,400원 → 33,500원).
   Google SA의 '팝업스토어 대행' 키워드 그룹 CPC가 41% 급등하며 전체 비용 증가를 주도했습니다.
   해당 키워드의 입찰가 상한을 조정하고, 전환율이 유사하면서 CPC가 낮은 대체 키워드
   그룹을 확장하는 것을 권장합니다."

**길이 기준:**
- 인사이트 1건 = 최대 5문장 이내
- 전체 리포트 = 핵심 인사이트 3~5건 + 부가 관찰 사항

## 분석 보고서 구조 템플릿

    [1] 이번 주/월 한 줄 요약 (KPI 기준)
    [2] 전체 성과 현황 (LEVEL 0)
    [3] 매체별 분석 (LEVEL 1) — KPI 기준 상위/하위 매체 특정
    [4] 주요 이슈 딥다이브 (LEVEL 2~5) — 이상 신호 감지된 영역만
    [5] 액션 아이템 (즉시 / 검증 후 / 중장기)
    [6] 다음 주/월 체크 포인트
